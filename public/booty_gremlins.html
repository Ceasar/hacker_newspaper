Hacker Newspaper has minor stability issues due to flaws in the Hacker News RSS feed, as well as the limitations of this site itself (which is a fairly simple hack).

The problem should fix itself. The process which generates Hacker Newspaper will run again automatically within an hour. Most Hacker Newspaper stability issues disappear within that time frame, and personally I (Giles, the site's creator) read Hacker News way too freaking often anyway. I've probably already seen this and decided not to do anything about it.

However, the code is on GitHub if you want to fix it, and patch requests are always welcome (especially when made in topic branches). It's built on Ruby and Python and uses nothing unusual in either language. The slicehost branch is effectively the master branch, but other than that the repo is very standard.
